# Rating System — Backend Study Project

This project was developed with a focus on learning essential backend concepts, using a rating system as the main domain. The application allows managing ratings, comments, and filters, while exploring different architectural approaches and common patterns used in modern APIs.

## Objectives

- Practice relational data modeling
- Implement authentication and authorization
- Use different HTTP methods
- Work with cache and performance optimization
- Explore GraphQL alongside REST
- Apply API best practices

## Features

- Rating system
- Comments per item
- Filtering and sorting
- Score-based ranking
- JWT authentication
- Basic permission control
- Redis caching
- GraphQL integration
- CORS support
- Multiple HTTP methods usage

## Technologies Used

- SQL Server
- Redis (cache)
- JWT (authentication)
- GraphQL
- REST API
- CORS
- HTTP Methods: GET, POST, PUT, DELETE, HEAD, OPTIONS

## Educational Purpose

This project is not intended for production use, but rather to serve as a laboratory for experimenting with:

- Backend architecture
- Data modeling
- Cache and performance
- Basic security
- API patterns
- REST + GraphQL integration

Project created exclusively for study purposes and technical improvement.
